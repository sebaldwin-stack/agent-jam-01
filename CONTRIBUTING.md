# Contributing

Thank you for your interest in improving this project!

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button on GitHub to create your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/REPO_NAME.git
cd REPO_NAME
```

### 3. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/description-of-fix
```

### 4. Make Your Changes

- Follow the existing code style
- Add comments for complex logic
- Update documentation if you change functionality

### 5. Test Your Changes

[Add instructions for how to test your changes locally]

```bash
# Example test commands
```

### 6. Commit Your Work

Write clear, descriptive commit messages:

```bash
git add .
git commit -m "Add feature: short description"
```

### 7. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 8. Open a Pull Request

1. Go to the original repository
2. Click "Compare & pull request"
3. Fill in the template
4. Submit

## Pull Request Guidelines

- **One feature per PR** — Keeps reviews focused
- **Describe what changed** — Don't make reviewers guess
- **Link related issues** — Use "Fixes #123" if applicable
- **Tests welcome** — But not required for this jam

## Questions?

- Open an issue for bugs or feature requests
- Ping @yourhandle in the Heyron community if you need help
